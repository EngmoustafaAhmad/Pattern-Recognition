# Class Imbalance؟

يحدث Class Imbalance عندما يكون عدد البيانات في بعض الفئات أكبر بكثير من فئات أخرى.

مثال في Fraud Detection (كشف الاحتيال):

Transaction Type	Number
Normal	9900
Fraud	100

يعني:

Normal = 99%
Fraud = 1%

المشكلة هنا أن البيانات غير متوازنة.

لماذا هذه مشكلة؟

لو دربنا نموذجًا على هذه البيانات، قد يتعلم طريقة سهلة جدًا:

أن يتوقع دائمًا:

Normal

وبذلك يحصل على دقة عالية.

مثال:

لو لدينا 10000 معاملة:

9900 Normal

100 Fraud

إذا توقع النموذج دائمًا:

Normal

سيكون:

Accuracy = 99%

لكن في الحقيقة النموذج لم يكتشف أي عملية احتيال.

لهذا Accuracy وحدها ليست مقياسًا جيدًا في هذه الحالة.

الحلول (Solutions)
## 1️⃣ Oversampling

زيادة عدد بيانات الفئة الصغيرة (Minority Class).

مثال:

لدينا:

Normal = 9900
Fraud = 100

نقوم بتكرار بيانات Fraud حتى تصبح مثل Normal.

مثال بعد المعالجة:

Normal = 9900
Fraud = 9900

الميزة:

يجعل البيانات متوازنة.

العيب:

قد يؤدي إلى Overfitting لأننا نكرر نفس البيانات.

مثال طرق:

Random Oversampling

SMOTE

## 2️⃣ Undersampling

تقليل عدد بيانات الفئة الكبيرة (Majority Class).

مثال:

بدل:

Normal = 9900
Fraud = 100

نأخذ فقط:

Normal = 100
Fraud = 100

الميزة:

يجعل البيانات متوازنة بسرعة.

العيب:

نفقد الكثير من البيانات المفيدة.

## 3️⃣ Weighted Loss Functions

بدل تغيير البيانات، نغير طريقة حساب الخطأ (Loss).

نجعل الخطأ في الفئة الصغيرة أكبر أهمية.

مثال:

إذا أخطأ النموذج في:

Fraud

نحسب خطأ أكبر.

مثال أوزان:

Normal weight = 1
Fraud weight = 10

بهذا يصبح النموذج مهتمًا أكثر بالفئة الصغيرة.

الميزة:

لا نضيع بيانات.

يعمل جيدًا مع Neural Networks و SVM.

مثال بسيط للتوضيح

تخيل اختبار به:

990 سؤال سهل
10 أسئلة صعبة

لو الطالب أجاب فقط الأسئلة السهلة:

سيحصل على:

99%

لكن في الحقيقة لم يحل الأسئلة المهمة.

هذا بالضبط ما يحدث مع Class Imbalance.

خلاصة الفكرة

Class Imbalance = فئة قليلة + فئة كثيرة

المشكلة:

Model becomes biased toward the majority class

الحلول الأساسية:

Oversampling → زيادة الفئة الصغيرة

Undersampling → تقليل الفئة الكبيرة

Weighted Loss → إعطاء الفئة الصغيرة وزنًا أكبر
